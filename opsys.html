Chapter 3 Operating systems Terminology

Term Definition
operating system The main underlying system of a computer, controls and dictates most processes.
job queue The wait-for-execution system in batch processing
FIFO An asset management system: "First in, first out"
interactive processing A system that allows program execution through dialogue
time-sharing A feature that allows multiple users to use a single system, dividing the time up.
multiprograming A form of time-sharing that splits time into intervals
application software A program that runs on top of the OS, used to carry out a task
system software A program that works alongside or a part of the OS to manage resources
utility software Software used to maintain a computer
graphical user interface (GUI) A visual interface that allows users to perform actions without using the terminal
shell A program that allows one to interact with the system
kernel The "heart" or "core" of the OS: manages and runs the computer
device driver A program which is used to allow the computer to interact with a device
file manager A usually visual program that allows one to control files
memory manager Program that Manages Computer memory (usually RAM)
virtual memory "fictional" memory that allows the computer to operate as if it had that memory amount.
boot strapping The term for how a program gets started, most commonly used to refer to the OS
ROM Read-Only-Memory, often used to store important files that shouldn't be changed
deadlock When two or more process cannot function due to their resources being allocated to another process
spooling "Holding on to" data for later.
super user (or administrator) A user with authorization to use secure functions/features
multi-factor authentication A form of authentication that requires a secondary, external method in order to log in.
