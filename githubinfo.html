All about GitHub

  • Q. What is a version control system?
  • A. A software that automatically tracks any change made to files or code
  • Q. What is Git?
  • A. A common version control system developed by Linus Torvalds
  • Q. What is GitHub?
  • A. A website hosting service used to store Git Versions and Repositories.
  • Q. What is the difference between Git and GitHub?
  • A. Git is a version control software, GitHub is a website used to store and manage version controlled software
  • Q. Who started GitHub and how was it started?
  • A. Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, Scott Chacon. Made using Ruby
  • Q. What company owns it now?
  • A. Microsoft
  • Q. How much does a GitHub account cost?
  • A. Outside of premium memberships, they are free.
  • Q. What is the Octocat?
  • A. The Trademarked "Mascot" of github.

Git/GitHub terms:

  • Repository: The File/Folder where the Code(base) is stored.
  • Commit: The "sent" changes to a set of code.
  • Fork: A copy of a codebase to allow one to make seperate changes to it
  • Push: Uploading to a Github Repo
  • Pull Request: A request to a repository to introduce new/external code
  • Workflows: A process or "program" that allows one to automatically run a process/set of commands.
  • Issues: A tab/group where one can list any "Issues" with the code: bugs, missing features, etc.
  • Raw Button: Allows one to download/access code without formatting.
  • Blame Button: Allows one to automatically check every line of code to see who changed what.
  • EXTRA: Check: A workflow or function implemented to automatically test if a piece of code passes a defined test (I.E: Doesn't break UI)
