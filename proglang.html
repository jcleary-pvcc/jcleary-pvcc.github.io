Chapter 6 Programming Languages

Term Definition
Machine Language The actual "language" understood by a computer. Uses binary
debugging The Process where one seeks out errors/bugs in the code, and corrects them.
Assembler A program that converts assembly code into machine language
Assembly Language A "low level" language that lies close to machine language
Translator Converts a High Level language to another High level language
Compiler "Compiles" a high level language into a low level one, typically assembly
Interpreter Preforms a high level language on a line-by-line basis
Cross-Platform Software Software that works across multiple platforms
Natural Language language "natural" to a human. usually mixed with the term pseudocode
Formal Language a specifically defined language
Machine Independence A feature that doesn't depend on characteristics/features of a machine.
High-Level Programming Language A more user friendly programming language that operates further away from machine language, requiring another program (usually an interpreter or compiler) to run
Object-Oriented Programming (OOP) A type of programming language where the language centers on "objects" such as classes. Java is a notable example
Scripting Language A usually purpose built language meant to direct a computer to perform a task. An example is Structured Data Query Language (SQL)
Data Types
  • integer: A "Whole" number without decimals.
  • float: A number with less restrictions, allows decimals.
  • string: a "sequence of characters" which the language doesn't calculate.
primitive data type The basic, simplest, data types that are typically shared by all languages, such as the above.
data structure A system in a programming language that "Structures" data, such as a list or set
Python collection data structures
  • list: uses [ ], indexed, ordered, and changable. Values can be duplicated.
  • tuple: uses ( ), indexed, ordered, not changeable. Values can be duplicated.
  • set: uses { }, unindexed, unordered, unchangeable.
  • dictionary: also uses { }ordered, and changeable. Defined by a pair of "key : value", keys cannot be duplicated.
Function A "sub-program" that can be pre-defined, or defined by the user, which can be called by a program
Local Variable A variable that cannot be used outside of the function it was defined in
Global Variable A Variable that is defined in the program itself, and can be used anywhere
Source Program The Pre-Compiled program, not ready to be run.
Object Program A Compiled program, which can be run on a computer.
